Figure 4: Premature Action Rate + Hallucination Dumbbell
Layout: Two panels side by side with a shared y-axis (model names appear only on the left). The left panel is narrower (~1:1.7 ratio with the right).
Shared y-axis: Six models from top to bottom: Opus 4.6, Gemini 3 Pro, GPT-5.2, Haiku 4.5, Gemini 3 Flash, GPT-5 Mini. A solid horizontal divider line separates the top three (frontier tier) from the bottom three (efficient tier). Lighter dotted lines separate individual models. Each model row has two sub-rows with alternating background shading — the top sub-row is light lavender (#ede7f6) for IK/Baseline, the bottom is light grey (#eceff1) for KE.
Panel (a) — Premature Action Rate: Horizontal bar chart. Each model has two bars stacked vertically within its row: the top bar is purple (#6B3FA0) for IK condition, the bottom bar is dark grey (#4A5E6A) for KE condition. Bars extend from 0% to 100% on the x-axis. Percentage labels appear at the end of each bar. The data from Table I: under IK, all models show 100% premature action. Under KE, Opus 4.6 drops to 20%, Haiku 4.5 drops to 0%, others vary (Gemini 3 Pro stays 100%, GPT-5.2 stays 100%, Gemini 3 Flash stays 100%, GPT-5 Mini stays 60%).
Panel (b) — Hallucination Dumbbell: For each model row, there are two dumbbell plots on the same horizontal axis (0%–100%): one for feature hallucination (diamond marker ♦) and one for location hallucination (square marker ■). Each dumbbell has two connected dots — purple for IK, grey for KE — joined by a line. Green downward arrows annotate cases where KE reduced the hallucination rate. The x-axis label is "Hallucination Rate (Avg.)". Under IK, all models hallucinate features and location at 100%. Under KE, Opus 4.6 drops features to 20% and location to 80%, Haiku 4.5 drops both to 0%, others show partial or no improvement.
Legend: A single legend row at the top spanning both panels. Purple square = IK (Baseline), grey square = KE (Knowledge-Equalized). Diamond = Features, square = Location. Green arrow = Reduction under KE.

Figure 5: Hypothesis Composition + Retrieval-Reasoning Gap
Layout: Two panels side by side. Left panel is wider.
Panel (a) — Hypothesis Composition (Diverging Bar Chart): A horizontal diverging bar chart centered at zero. For each model (plus a "Ref. OA" reference row at the top), bars extend left for log-derived hypotheses (mean count per trial) and right for domain-grounded hypotheses (mean count per trial). There are four bar colors total: log-derived IK, log-derived KE, domain-grounded IK, domain-grounded KE. The x-axis shows "Mean Hypothesis Count" with negative values on the left (log-derived) and positive on the right (domain-grounded). Reference OntoAgent shows 1.0 log-derived and 2.0 domain-grounded. Key data points: GPT-5 Mini under KE produces 7.0 domain-grounded hypotheses (the highest). Under IK, most models produce primarily log-derived hypotheses with minimal domain-grounded ones.
Panel (b) — Retrieval-Reasoning Gap (KE Only): For each model, four horizontal metrics are shown: "Fetch Queried" (did the model call FETCHPLAN for the fetch procedure), "Fetch Followed" (did it actually follow the retrieved procedure), "Diagnose Queried" (did it call FETCHPLAN for the diagnostic procedure), "Diagnose Followed" (did it follow the diagnostic procedure). Each metric is a percentage bar from 0%–100%. The critical finding visualized here: Opus 4.6 queried the fetch procedure 80% of trials but followed it 0%. It queried diagnose 100% but followed it 0%. GPT-5 Mini never queried either procedure yet still improved. A "Gap" annotation highlights the difference between querying and following.
